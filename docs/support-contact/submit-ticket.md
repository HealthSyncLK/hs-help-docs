# Submit a Support Ticket

Create a support ticket for detailed assistance.

## When to Submit a Ticket

### Best For
- Complex technical issues
- Problems requiring investigation
- Bug reports
- Feature requests
- Billing disputes
- Issues with detailed history
- Non-urgent support needs
- Outside live chat hours

### vs. Live Chat
Use tickets when:
- ‚úÖ Need detailed explanation
- ‚úÖ Multiple screenshots required
- ‚úÖ Problem needs investigation
- ‚úÖ Want written documentation
- ‚úÖ Not urgent

Use chat when:
- Quick question
- Need immediate help
- Simple troubleshooting
- During business hours

## What to Include

### Required Information

**Subject Line**
- Clear and concise
- Describes the issue
- Example: "Cannot upload PDF files"

**Description**
- What you were trying to do
- What actually happened
- When it started
- How often it occurs
- What you've tried already

**Account Information**
- Email address (auto-filled if logged in)
- Account type (Free, Plus, Premium)
- Affected family members (if applicable)

### Helpful Additional Information

**Error Messages**
- Exact text of error
- Screenshot of error
- Error codes

**Browser/Device Info**
- Browser name and version
- Operating system
- Mobile device model
- App version

**Steps to Reproduce**
1. First, I did this...
2. Then, I clicked here...
3. The error appeared when...

**Screenshots**
- Error messages
- Unexpected behavior
- Relevant screens
- Console logs (if technical)

### Attachments

You can attach:
- Screenshots (PNG, JPG)
- Screen recordings (MP4, MOV)
- Log files
- Sample documents (if issue related)
- Error reports

Limits:
- Max 5 files per ticket
- 10MB per file
- 50MB total

## Ticket Lifecycle

### 1. Submission
- Ticket created
- Unique number assigned (HS-XXXXXX)
- Confirmation email sent
- Auto-reply with ticket number

### 2. Assignment
- Routed to appropriate team
- Agent assigned
- Ticket reviewed
- Initial research conducted

### 3. First Response
- Agent replies with:
  - Acknowledgment
  - Questions for clarification
  - Initial troubleshooting steps
  - Estimated timeline

### 4. Investigation
- Agent investigates issue
- May request more information
- Tests reproduction steps
- Consults specialists if needed

### 5. Resolution
- Solution provided
- Steps to resolve explained
- Confirmation requested
- Follow-up planned

### 6. Closure
- Issue confirmed resolved
- Satisfaction survey sent
- Ticket closed
- Can reopen within 7 days

## Tracking Your Ticket

### Email Updates

You'll receive emails when:
- Ticket created
- Agent responds
- Status changes
- Resolution provided
- Ticket closed

### Ticket Portal

View your tickets:
1. Visit [support.healthsync.com](https://support.healthsync.com)
2. Log in
3. See all your tickets
4. View status and updates
5. Add comments
6. Upload additional files

### Ticket Status

- **New**: Just created, not yet assigned
- **Open**: Agent is working on it
- **Waiting**: Awaiting your response
- **On Hold**: Awaiting internal resource
- **Resolved**: Solution provided
- **Closed**: Issue confirmed resolved

## Responding to Tickets

### When Agent Needs Info

**Email:**
- Reply directly to ticket email
- Keep ticket number in subject
- Inline responses preferred
- Attach files as needed

**Portal:**
- Log in to support portal
- Find your ticket
- Add comment
- Upload files
- Save

### Response Time Matters

If you don't respond:
- After 3 days: Reminder sent
- After 7 days: Ticket marked "Waiting on Customer"
- After 14 days: Ticket auto-closed
- Can reopen if needed

## Escalating Tickets

### When to Escalate

- No response within SLA
- Resolution not satisfactory
- Issue remains unresolved
- Need management attention
- Compliance concerns

### How to Escalate

**Reply to ticket:**
"Please escalate this ticket"

**Or email:**
escalation@healthsync.com
Include ticket number

**Premium customers:**
Contact account manager directly

### What Happens

- Reviewed by senior support
- Manager assigned
- Priority increased
- Faster resolution track
- Follow-up call scheduled (Premium)

## Ticket Best Practices

### ‚úÖ Do
- Provide complete information upfront
- Include screenshots
- List steps to reproduce
- Be specific and detailed
- Respond promptly to agent questions
- Confirm when resolved
- Rate your experience

### ‚ùå Don't
- Create duplicate tickets
- Demand immediate resolution
- Reply to unrelated old tickets
- Send multiple emails to different addresses
- Include password or payment details
- Be abusive to support staff

## Multiple Related Issues

### Option 1: Single Ticket
If issues are related:
- List all in one ticket
- Agent addresses each
- Keeps information together
- Easier tracking

### Option 2: Separate Tickets
If issues are unrelated:
- Create separate tickets
- Different teams can work in parallel
- Faster resolution
- Cleaner tracking
